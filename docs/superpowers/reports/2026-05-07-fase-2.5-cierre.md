### Queries que siguen MAL (7/25) — diagnóstico vía debug tools
1. **Q1 "Nike Air Max 270 talle 42"** → empty. Hipótesis: el mock generó productos pero falló enrichment.
2. **Q3 "Samsung Galaxy S24 Ultra"** → devuelve iPhones. Hipótesis: **semantic cache θ=0.92 está reusando el resultado de Q2 "iPhone 15"** porque los embeddings son cercanos.
3. **Q4 "Sony WH-1000XM5"** → idem (cache semantic con Q2).
4. **Q6 "audífonos bluetooth"** → iPhones. Idem.
5. **Q7 "bocinas portátiles"**, **Q10 "auriculares para correr"** → idem.
6. **Q14 "juguete educativo para niño 5 años"**, **Q25 "juguetes bebé"** → empty. Hipótesis: similar a Q1.
7. **Q15 "vestido para boda femenino"** → Nike Air Max Mujer (no vestido). Hipótesis: filter `gender_target=femenino` matched on Nike Air Max Mujer producto (de Q5) en lugar de generar vestidos vía mock.

**El bug raíz #3 que F2.5 no atacó: θ=0.92 es muy permisivo.** Master doc Sec 9 dejó θ a calibrar empíricamente en Fase 5 — pero ya tenemos evidencia ahora. Posibilidad: subir θ a 0.95 o 0.97. Esto va a Fase 5 (calibración) o se puede tunear ahora.

**El bug #4: queries que vuelven empty.** Posible causa: cuando smart mock se invoca con `category` que el LLM no popula consistentemente, los productos generados pueden no matchear ni BM25 ni cosine después de enrichment. Inspeccionable con `pnpm explain "Nike Air Max 270 talle 42"`.

## 6. Riesgos vivos identificados durante la ejecución

1. **Semantic cache θ=0.92 es permisivo para queries con marcas distintas.** Q3 "Samsung" devuelve iPhone porque el embedding es similar a Q2 "iPhone". → Fase 5 calibración, o tune manual ahora.
2. **Smart mock fields opcionales.** DeepSeek no siempre popula `source_product_id`/`brand`/`description` — F2.5 los hace opcionales y autogenera `source_product_id` cuando falta. Funcional, pero brand vacío puede afectar filtros futuros.
3. **Empty results en algunas queries** (Q1, Q14, Q25). Posible: smart mock genera N productos pero enrichment falla silencioso. Inspeccionable con `pnpm explain` en próxima iteración.
4. **Costo del smart mock en producción**. Cada mock fallback ahora cuesta ~$0.0005 (DeepSeek) en vez de ~$0 (fixture). Sigue siendo 80× más barato que el agregador real ($0.04). OK.
